Ruby for C# Programmers

There are plenty of good books on the Ruby programming language. However many of the concepts required to program in Ruby are identical or have analogies in C#. If you already know c# then, lots of stuff doesn't need to be explained to you or could be quickly explained simply by saying this is Ruby's equivalent of Foo in c#/.NET or "these work just like in c# except a, b and c".

This is mainly a personal diary for me while I'm learning Ruby, but there is some small chance that it might help other c# devs out there who are also jumping ship.

Structure and Execution

Comments

Feature C# Ruby
Single line comment // #
Multi-line comment /* I'm a c# multi line comment */ =begin I'm a ruby multi line comment =end Typically Ruby just prefixes multiple lines with #. The =begin..=end syntax is actually designed to be used for embedding documents but could technically be used for multi-line comments in the same way as /*...*/ in c#.

Identifiers

Feature C# Ruby
Constants const FOO_BAR = 5; FooBar = 5 In ruby, identifiers starting with capital letters are implicitly constants.
Variables var fooBar = 5; C# identifiers can be prefixed with @ signs and can contain unicode escape sequences. foo_bar = 5 The ruby convention is to separate multiple words with an underscore.

C# identifiers can be prefixed with @ signs and can contain unicode escape sequences. Ruby identifiers can contain actual unicode characters. Additionally, in ruby:

  • $ prefixes global variables
  • @ prefixes instance variables
  • @@ prefixes class variables
  • ? suffixes methods that return a boolean
  • ! suffixes methods that alter their parameters (like ref/out parameters in c#)
For example:

# A global variable @data # An instance variable @@counter # A class variable empty? # A Boolean-valued method or predicate sort! # In-place alternative to regular sort timeout= # A method invoked by assignment

Data

Feature C# Ruby
Data resource files __END__ When on a line by itself, this keyword marks the end of the program (and the beginning of a data section)

Whitespace

Feature C# Ruby
statement terminator ; ;

Only required in ruby when more than one statement appears on a single line. If the code on a single line in ruby is syntactically complete then the new line is implicitly the statement terminator.

Ruby program behave differently depending on where you insert a newline. For example, the following code adds x and y and assigns the sum to total: total = x + # Incomplete expression, parsing continues y But this code assigns x to total, and then evaluates y, doing nothing with it: total = x # This is a complete expression + y # A useless but complete expression

Data types

Strings

Feature C# Ruby
String literal "Hello" 'World'
Format string var target = "World"; string.Format("Hello {0}!", target); target = 'World!' "Hello #{target}!" Ruby's equivalent is called interpolation and is a language feature rather than a method. Unlike the string.Format method in c#, interpolated strings in ruby don't need any explicit arguments - any valid ruby expression can be used in a string interpolation.

Numbers

Feature C# Ruby
Integer literals 1000000000 1_000_000_000 Ruby allows underscores in integer literals, for readibility
Bases 0xFF Convert.ToInt32("0377", 8); Convert.ToInt32("11111111", 2); c# doesn't natively support bases other than 10 and 16 - the Convert helper needs to be used for other bases 0xFF # 255 in hexadecimal 0377 # Octal 0b1111_1111 # Binary In ruby, an integer literal starting with a zero and consisting of more than one digit will be interpreted in some base other than 10.
